# Результаты производительности (Framework Hibernate)

**Тестовые данные:** 10,000+ записей в таблице points
**База данных:** H2 (in-memory)

| Операция | Время (ms) |
|----------|------------|
| SELECT * FROM points | 77 |
| JOIN functions + points | 18 |
| INSERT point | 55 |
| UPDATE point | 49 |
| DELETE point | 24 |

## Выводы
1. Hibernate добавляет накладные расходы на маппинг объектов
2. JPQL запросы удобнее для сложных операций
3. Автоматическое управление транзакциями и связями
