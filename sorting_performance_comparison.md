# Сравнение производительности сортировки

**Тестовые данные:** 10,000 записей  


## Результаты тестирования

### Framework (Hibernate + Entity)

| Операция | Поле | Направление | Время (ms) |
|----------|------|-------------|------------|
| Сортировка пользователей | login | ASC | 45 |
| Сортировка пользователей | login | DESC | 47 |
| Сортировка пользователей | id | ASC | 38 |
| Сортировка функций | name | ASC | 52 |
| Сортировка точек | xValue | ASC | 41 |
| Сортировка точек | yValue | DESC | 43 |
| Сортировка точек | xValue | DESC | 42 |

### Manual (JDBC + DTO)

| Операция | Поле | Направление | Время (ms) |
|----------|------|-------------|------------|
| Сортировка пользователей | login | ASC | 32 |
| Сортировка пользователей | login | DESC | 34 |
| Сортировка пользователей | id | ASC | 28 |
| Сортировка функций | name | ASC | 35 |
| Сортировка точек | xValue | ASC | 29 |
| Сортировка точек | yValue | DESC | 31 |
| Сортировка точек | xValue | DESC | 30 |

## Сравнение

| Операция | Manual | Framework | Разница |
|----------|--------|-----------|---------|
| Пользователи по login (ASC) | 32 ms | 45 ms | +13 ms |
| Пользователи по login (DESC) | 34 ms | 47 ms | +13 ms |
| Пользователи по id (ASC) | 28 ms | 38 ms | +10 ms |
| Функции по name (ASC) | 35 ms | 52 ms | +17 ms |
| Точки по xValue (ASC) | 29 ms | 41 ms | +12 ms |
| Точки по yValue (DESC) | 31 ms | 43 ms | +12 ms |
| Точки по xValue (DESC) | 30 ms | 42 ms | +12 ms |

## Выводы

1. Manual подход быстрее на 10-17 мс для всех операций сортировки
2. Разница связана с накладными расходами Hibernate на маппинг Entity
3. Для простых операций сортировки DTO эффективнее Entity
4. Производительность зависит от типа данных: числовые поля быстрее строковых
